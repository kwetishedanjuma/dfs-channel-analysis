# Configuration Files

System-specific configuration files containing data paths, parameters, and settings for each DFS analysis.

## Purpose
- Centralise system-specific parameters
- Maintain consistent analysis settings
- Enable reproducible analysis across systems

## Configuration Parameters
- Data file paths and naming conventions
- System boundaries and domain divisions
- Analysis parameters and thresholds
- Output settings and file locations

## Usage
Import configuration files into main analysis scripts to ensure consistent processing across all five DFSs.
